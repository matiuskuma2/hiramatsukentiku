### 初期導線（ログイン後フロー）

```
1. ユーザーがドメインにアクセス
2. Cloudflare Access がJWTを発行（メール認証 or IdP認証）
3. アプリが CF-Access-Authenticated-User-Email ヘッダーを取得
4. GET /api/users/me を呼出し
   ├── 200: app_users に登録済み＆active → DASHBOARD にリダイレクト
   │         last_login_at を更新
   ├── 403: 未登録 or inactive → ERROR_403 ページ表示
   └── (開発環境): DEV_USER_EMAIL バイパスでスキップ

5. DAシボード到着後:
   ├── ロール=admin/manager: 全メニュー表示
   ├── ロール=estimator: 管理者メニュー非表示
   └── ロール=viewer: 編集系ボタン非表示、閲覧のみ
```

### 画面 1: LOGIN（ログイン）

| 項目 | 内容 |
|------|------|
| **画面ID** | LOGIN |
| **URL** | `/login` |
| **目的** | Cloudflare Access 経由の認証入口。直接的なUIは CF Access が提供 |
| **使用者** | 全員（認証前） |
| **主要表示項目** | CF Access 認証画面（メール入力 or IdP選択） |
| **更新対象テーブル** | なし（CF Access側で処理） |
| **使用API** | なし（CF Access が JWT を発行） |
| **権限** | public |
| **遷移元** | ドメイン直アクセス |
| **遷移先** | 認証成功 → DASHBOARD（登録済み）/ ERROR_403（未登録）|

#### 画面動作

```
1. ユーザーがシステムURLにアクセス
2. CF Access がメール認証画面を表示
3. メール入力 → ワンタイムコード発行 → 認証
4. 認証成功: JWT がヘッダーに付与され、アプリへリダイレクト
5. アプリ側 auth.ts middleware が:
   a. CF-Access-Authenticated-User-Email を取得
   b. app_users で照合
   c. active かつ登録済み → c.set('user', {...}) → DASHBOARD
   d. 未登録/inactive → ERROR_403
```

| | 「アーカイブ」ボタン | admin/manager のみ | 確認ダイアログ表示 |

#### 楽観ロック動作

```
1. 画面表示時: GET /api/projects/:id → version を保持
2. 保存時: PUT /api/projects/:id + { ...data, version: N }
3. サーバー:
   a. version一致 → 更新成功, version = N+1
   b. version不一致 → 409 Conflict
4. 409時のUI:
   ┌──────────────────────────────────────────┐
   │ ⚠ データが他のユーザーにより更新されました │
   │                                          │
   │ 最新のデータを読み込み直してください。     │
   │                                          │
   │            [リロードする]                 │
   └──────────────────────────────────────────┘
```

## APIパラメータ仕様

### ページネーション共通パラメータ

| パラメータ | 型 | デフォルト | 説明 |
|----------|-----|----------|------|
| `page` | INTEGER | 1 | ページ番号 |
| `per_page` | INTEGER | 20 | 1ページあたりの件数（最大100） |
| `sort` | TEXT | `created_at` | ソート対象カラム |
| `order` | TEXT | `desc` | ソート順（`asc` / `desc`） |

### 楽観ロック対応

PUT/POSTリクエスト時に `version` パラメータを含める。サーバー側で現在のバージョンと比較し、不一致の場合は **409 Conflict** を返す。

```json
// リクエスト
PUT /api/projects/1
{
  "project_name": "佐藤邸（更新）",
  "version": 3
}

// 409レスポンス
{
  "error": "conflict",
  "message": "データが他のユーザーにより更新されています。画面をリロードしてください。",
  "current_version": 4
}
```
