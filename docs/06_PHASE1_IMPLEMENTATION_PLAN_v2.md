# Phase 1 実装計画・優先順位 v2（改訂版）

> **改訂履歴**: 07_CROSS_REVIEW_AND_RESOLUTIONS.md の指摘事項を反映。リスク対策を具体化し、工数見積もりを修正。

## Phase 1 の定義

**ゴール：** 現行スプレッドシートと同じ条件を入れたら、同じ工種が出て、同じ金額が出て、どこを人が確認すべきかも分かる状態にする。

**制約：**
- Cloudflare Pages + Workers + D1
- OpenAI APIは条件チェック機能のみPhase1に含む
- 認証は Cloudflare Access（Zero Trust）
- 対象ユーザーは社内5〜10名

**v2で追加された設計要件：**
- マスタ系テーブルは TEXT PK（シードID体系をそのまま使用）
- `cost_master_item_versions` テーブルによるバージョン管理
- `cost_rule_conditions` のJSON構造対応
- 楽観ロック、案件コード自動採番
- 12種類の計算方式対応（`per_piece` 追加）
- final_* 値の算出ロジック明文化

**計算フロー（v2改訂版）：**
```
1. 案件条件を読み込む
2. 第1パス: 全工種を独立計算
   2a. 工種内の全明細マスタをループ
   2b. 各明細に対して:
     - JSON構造のルール条件を評価
     - 採用/非採用を決定
     - 計算方式に応じて数量・金額を算出
     - cost_master_item_versions から現在有効バージョンをスナップショット
     - base_unit_price / base_fixed_amount を正しく使い分け
   2c. project_cost_items に保存（100件バッチ分割）
3. 第2パス: 工種間連動ルールを適用
   - rule_group = "cross_category" のルールを評価
4. final_* 値を算出（T-01ロジック適用）
5. 工種ごとの合計を project_cost_summaries に保存
6. 警告条件を評価 → project_warnings に保存
7. 全体合計 + 粗利率を算出
```

**v2での追加タスク（+1〜2日）：**

## リスクと対策（改訂版）

### リスク1：スプレッドシートの計算ロジックが一部不明
- **対策：** 不明箇所は担当者にヒアリング。暫定値で実装し、後で調整。
- **該当：** 足場工事、建材・副資材の詳細が未提出。
- **v2追加対策：** 未提出シートはPhase 1スコープから除外し、手入力のみ対応。

### リスク2：マスタデータの網羅性
- **対策：** Priority A の10工種を最優先で完全投入。残り27工種は段階的に追加。
- **v2追加対策：** シードファイルのJSON→SQL変換を自動化し、追加投入の工数を削減。

### リスク3：条件分岐の複雑さ
- **対策：** Step 4の計算エンジンは十分な時間を確保（6-8日）。テスト駆動で実装。
- **v2追加対策：** JSON構造のルール評価エンジンは汎用的に設計し、新ルール追加が設定変更のみで完了する構造にする。

### リスク4：現場との数字不一致
- **対策：** Step 11で必ずスプレッドシートとの突合を行う。差異は1件ずつ解消。
- **v2追加対策：** 05_MASTER_DATA_PLAN_v2.md のルール発火テスト11パターンを全て実施。

### リスク5：D1のパフォーマンスとバッチ制限
- **対策：** 100件バッチ分割を実装。トランザクション内で連続実行。
- **v2追加対策：** 個別INSERTへのフォールバック機構を用意。

### リスク6（v2追加）：楽観ロック衝突
- **対策：** 409 Conflict のUI対応（リロード促進メッセージ）。Phase 1の利用者数では低頻度。

### リスク7（v2追加）：バージョン管理の複雑性
- **対策：** Phase 1では「現在有効バージョン」のみを使用。過去バージョンの参照UIはPhase 2。

## 成功基準（改訂版）

Phase 1 が「完成」と言える条件：

1. **全37工種の原価入力が可能**（自動/半自動/手動のいずれかで）
2. **Priority A工種17種が自動計算可能**（うち10種はシードデータ完備）
3. **テスト案件3パターンでスプレッドシートと数字が一致**
4. **手修正時に理由が記録される**（`override_reason` 必須バリデーション）
5. **条件漏れ警告が表示される**
6. **マスタ変更・案件変更の履歴が残る**（バージョン管理含む）
7. **粗利率・想定売価がサマリーで見える**（3グループ: 標準/太陽光/オプション）
8. **楽観ロックが機能する**（同時編集時に409エラーが返る）
9. **案件コードが自動採番される**（`YYYY-NNN` フォーマット）
10. **スプレッドシート5フィールドが完全に保持されている**
